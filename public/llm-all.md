# 能力总览

本文用于帮助首次接触 DocMee Open Platform 的团队先理解平台的能力边界，再判断应该优先走 `UI 接入`、`API 接入` 还是 `开放能力`。

## 平台能做什么

- 通过 `UI 接入` 把完整的 AI PPT 创作、模板、编辑和下载体验嵌入到业务系统
- 通过 `API 接入` 控制任务创建、内容生成、模板处理、PPT 下载与管理链路
- 通过 `开放能力` 获取特定能力模块，例如 HTML 转 PPTX 这类单点能力
- 通过模板、PPT、记录和定价相关页面补齐完整的交付与运营链路

## 能力边界

DocMee 更适合承担“演示内容生产平台”角色，而不是单纯的静态模板下载站。它擅长的是：

- 让业务系统快速获得 AI PPT 创作能力
- 提供可编排的 PPT 生成链路和中间结果
- 支持模板、文件、记录、下载等配套能力

如果你的目标只是做一个完全自定义的前端编辑器，或者只需要少量静态模板展示，DocMee 并不是最直接的选择。

## 如何选择接入方式

### 什么时候选 UI 接入

优先选择 [UI 接入](/ui-integration) 的场景：

- 你希望快速嵌入完整工作台，而不是自己搭建整套前端体验
- 你需要创作页、列表页、编辑页和模板页的一体化能力
- 你更关注集成速度和完整体验，而不是逐个控制底层接口

### 什么时候选 API 接入

优先选择 [快速开始](/getting-started) 和 [API 参考](/api-reference) 的场景：

- 你需要在服务端完全控制任务链路和中间结果
- 你需要把生成能力编排进现有业务流程
- 你需要按接口粒度管理模板、PPT、记录和下载逻辑

### 什么时候选开放能力

优先选择 [开放能力](/open-capabilities) 的场景：

- 你只需要某个特定能力，而不是整套平台能力
- 你希望从专项能力快速验证可行性
- 你要处理类似 [HTML 转 PPTX](/open-capabilities/html-to-pptx/introduction) 这样的单点能力

## 推荐阅读顺序

1. 先阅读本页，确定你的目标更接近 `UI 接入`、`API 接入` 还是 `开放能力`
2. 要嵌入完整工作台时，进入 [UI 接入](/ui-integration)
3. 要从后端接口联调时，进入 [快速开始](/getting-started)
4. 要查看所有接口分类时，进入 [API 参考](/api-reference)
5. 要评估专项能力时，进入 [开放能力](/open-capabilities)

## 下一步

- 直接开始嵌入：阅读 [UI 接入](/ui-integration)
- 跑通 API 主链路：阅读 [快速开始](/getting-started)

# 创建接口 Token


## 获取您的 Api-Key

您可以前往[docmee开放平台](https://open.docmee.cn/)注册账号，并获取您的 Api-Key。

**获取方式**

1. 登录 [docmee开放平台](https://open.docmee.cn/)
2. 点击右上角 "进入工作台" 按钮
3. 在“我的工作台”页面，选择 “API KEY” 标签页
4. 在“API KEY”页面，点击“复制”




## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/api/user/createApiToken">https://open.docmee.cn/api/user/createApiToken</InlineCode>
- 鉴权：请求头传 `Api-Key`

## 接口作用

创建后续 API 调用使用的临时 `token`。该 Token 支持按 `uid` 做数据隔离，也支持通过 `limit` 限制最多可生成的 PPT 次数。

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/api/user/createApiToken \
  --header "Content-Type: application/json" \
  --header "Api-Key: YOUR_API_KEY" \
  --data '{
  "uid": "third-party-user-id",
  "limit": 10,
  "timeOfHours": 2
}'
```

## 请求参数

```json
{
  "uid": "third-party-user-id",
  "limit": 10,
  "timeOfHours": 2
}
```

- `uid`：第三方用户 ID，建议传入，便于用户隔离
- `limit`：允许生成 PPT 的次数；`0` 表示不能生成
- `timeOfHours`：有效期小时数，默认 2，最大 48


## 请求字段说明

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `uid` | `string` | 否 | `third-party-user-id` | 第三方用户标识，用于隔离数据。 |
| `limit` | `number` | 否 | `10` | 允许调用或生成的次数上限。 |
| `timeOfHours` | `number` | 否 | `2` | 字段用途请结合接口场景传入。 |

## 响应示例

```json
{
  "data": {
    "token": "sk_xxx",
    "expireTime": 7200
  },
  "code": 0,
  "message": "操作成功"
}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `data` | `object` | `{"token":"sk_xxx","expireTime":7200}` | 业务数据对象。 |
| `data.token` | `string` | `sk_xxx` | 字段用途请结合接口场景传入。 |
| `data.expireTime` | `number` | `7200` | 字段用途请结合接口场景传入。 |
| `code` | `number` | `0` | 通用响应状态码，0 表示成功。 |
| `message` | `string` | `操作成功` | 服务端返回的结果说明。 |

## 接口作用

```bash
  "scene": "产品介绍",
```


```bash
  "questionMode": false,

  "length": "medium",
```

# AI 智能布局排版

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/api/ppt/v2/generatePptxByAi">https://open.docmee.cn/api/ppt/v2/generatePptxByAi</InlineCode>

## 接口作用

根据任务 ID 和 `JSON` 结构化大纲发起 AIL 智能布局任务，按页流式生成 `HTML` 结果。

<ImportantNote title="返回方式">

    <InlineCode>latestData</InlineCode> 做恢复。

| `templateId` | `string` | 否 | `template_xxx` | 模板 ID，用于给 AIL 增加模板约束。 |

## 参数说明

- `data` 需要是完整的结构化大纲对象，不建议传未完成的流式片段。

## 响应字段说明

| --- | --- | --- | --- | --- |
| `id` | `string` | 否 | `task_xxx` | 业务对象 ID。 |
| `stream` | `boolean` | 否 | `true` | 是否启用流式返回。 |
| `markdown` | `string` | 否 | `# 主题 / ## 章节 / ### 页面标题 / #### 内容标题 / - 内容` | Markdown 格式的内容结构。 |

## 响应方式

支持 `event-stream` 和 `application/json`，整体结构与 [`generateContent`](/api-reference/v2/generate-content) 一致。

## 你可以用 API 做什么

- 根据主题、文档、网页、粘贴文本或 Markdown 生成 PPT
- 流式生成大纲与内容，支持边生成边展示
- 按模板风格生成或替换 PPT
- 下载、保存、编辑和管理已生成的 PPT
- 管理自定义模板、查询积分与调用记录

## 推荐接入路径

如果你是首次接入，推荐优先使用 `V2`：

1. 调用 [`createApiToken`](/api-reference/auth/create-api-token) 获取调用令牌
2. 调用 [`createTask`](/api-reference/v2/create-task) 创建生成任务
3. 调用 [`generateContent`](/api-reference/v2/generate-content) 生成大纲与内容
4. 调用 [`generatePptx`](/api-reference/v2/generate-pptx) 生成最终 PPT
5. 按需调用模板、下载、保存等后续接口

完整示例见 [生成第一份 PPT](/getting-started/first-ppt)。如果你接入的是 `JSON` 大纲 + 页面级智能排版流程，请阅读 [AI 智能布局](/getting-started/ail-ppt)。

## 接口调用方式

所有接口都需要鉴权，主要有两种方式：

- 服务端先用 `Api-Key` 调用 [`createApiToken`](/api-reference/auth/create-api-token)，再把返回的 `token` 用于后续接口
- 服务端直连场景下，也可直接在部分接口中使用 `Api-Key`

后续 API 一般通过以下方式鉴权：

- 请求头传 `token: xxx`
- 封面图等资源地址通过 `?token=xxx` 访问

## V2 与 V1 的关系

- `V2` 是当前官方推荐方案，任务链路更清晰，适合新接入
- `V1` 仍然保留，适合历史系统兼容或特定调用模式
- 本文档会保留全部 `V1` 接口，并在 API 参考中明确标注兼容属性

# 生成第一份 PPT （简易版）


这一节介绍便携版接口。它是一套新的上层 API，和原有 `docmee.cn` 标准版接口分开部署，建议的接入地址如下：

```text
https://open.docmee.cn/v2/api/ppt/portable
```

便携版的目标是把标准版里的“创建任务 + 生成内容 + 生成 PPT + 获取下载地址”封装成更少的接口，适合直接接收用户输入并输出最终 PPT 的场景。

这个流程里，业务侧只需要关心两件事：

- 创建任务时传入用户输入、模板 ID 和生成参数
- 根据 `stream` 选择“轮询结果”或“SSE 持续接收结果”

<Callout type="info">
如果你需要完整控制每一个中间步骤，请使用[完整版](/getting-started/template-mode/1_complete-version)。
</Callout>

## 便携版接口设计

便携版保留 2 种获取结果的方式：

- 异步任务轮询：创建任务后，轮询结果接口获取最终 PPT
- SSE 长连接：创建任务时传 `stream=true`，持续接收生成进度和最终结果

对应只需要 2 个接口：

- [`POST /api/ppt/portable/create-task`](/api-reference/portable/create-task)
- [`POST /api/ppt/portable/task-result`](/api-reference/portable/task-result)

## 第 1 步：创建调用 Token

便携版仍然沿用现有鉴权方式。先在服务端使用 `Api-Key` 调用 [`createApiToken`](/api-reference/auth/create-api-token)。

拿到返回的 `token` 后，后续请求都在 Header 中传入：

```http
token: YOUR_TEMP_TOKEN
```

## 第 2 步：创建便携版任务

调用 [`POST https://open.docmee.cn/v2/api/ppt/portable/create-task`](/api-reference/portable/create-task) 创建任务。

### 请求方式

创建任务支持两种请求体格式：

| Content-Type | 适用场景 |
| --- | --- |
| `application/json` | 不需要上传文件时，直接传文本参数 |
| `multipart/form-data` | 需要上传参考文件时，使用 `file` 字段 |

如果当前场景需要上传文件，必须使用 `multipart/form-data`；如果不需要上传文件，推荐使用更简单的 `application/json`。

### 请求参数

便携版的参数设计以 [`createTask`](/api-reference/v2/create-task) 和 [`generateContent`](/api-reference/v2/generate-content) 的已有能力为基础，只保留上层封装真正需要的字段。




| 参数 | 类型 | 必填 | 来源 | 说明 |
| --- | --- | --- | --- | --- |
| `type` | `1-7` |✅| [`createTask`](/api-reference/v2/create-task) | 任务类型。`1` 智能生成；`2` 上传文件；`3` 思维导图；`4` Word 精准转 PPT；`5` 网页链接；`6` 粘贴文本；`7` Markdown 大纲 |
| `content` | `string` | ❌ | [`createTask`](/api-reference/v2/create-task) | 根据 `type` 传主题、网页链接、文本内容或 Markdown |
| `file` | `File[]` | ❌ | [`createTask`](/api-reference/v2/create-task) | 上传文件时使用，仅 `multipart/form-data` 支持 |
| `templateId` | `string` | ✅ | 便携版新增 | 指定最终生成 PPT 使用的模板 ID |
| `stream` | `boolean` | ❌ | [`generateContent`](/api-reference/v2/generate-content) | 是否使用 SSE 长连接返回生成过程，默认 `false` |
| `length` | `short \| medium \| long` | ❌ | [`generateContent`](/api-reference/v2/generate-content) | 内容长度偏好 |
| `scene` | `string` | ❌ | [`generateContent`](/api-reference/v2/generate-content) | 演示场景，例如“产品介绍” |
| `audience` | `string` | ❌ | [`generateContent`](/api-reference/v2/generate-content) | 受众类型，例如“客户” |
| `lang` | `string` | ❌ | [`generateContent`](/api-reference/v2/generate-content) | 输出语言，例如 `zh`、`en` |
| `prompt` | `string` | ❌ | [`generateContent`](/api-reference/v2/generate-content) | 附加要求，建议保持简短 |
| `useAgent` | `boolean` | ❌ | - | 是否使用智能体生成，默认 `false` |


`useAgent` 仅在任务类型为 `1` 时生效，其他任务类型会忽略该字段。

`useAgent` 为 `true` 时，会使用智能体（智能体会自行思考、搜索、总结）生成内容。


<Callout type="info">
如果您想让大纲效果更好，您可以开启智能体生成，开启后智能体会自行思考、搜索、总结（内容生成更准确，但耗时更长，消耗1积分）。

| 场景 | 建议传参 |

## 简易版与完整版的区别

简易版适合下面这类场景：

- 你只想接收用户输入，然后拿到最终 PPT 结果
- 你希望直接使用指定模板生成成品
- 你希望前端逻辑尽量简单，既可以轮询，也可以直接接 SSE

如果你需要以下能力，请使用[完整版](/getting-started/template-mode/1_complete-version)：

- 先生成 Markdown，再让用户手动编辑内容
- 对内容生成和 PPT 生成做分步控制
- 在各个阶段插入自定义审核、编辑或重试逻辑

## 常见建议

- 没有文件上传时优先用 `application/json`
- 需要上传文件时使用 `multipart/form-data`
- 建议业务侧保存 `taskId`，用于断线恢复和失败重试
- `templateId` 建议通过[`templates`](/api-reference/templates/list-templates)提前获取
- 生产环境由服务端创建 Token，不要在前端暴露 `Api-Key`

# 能力总览

本文用于帮助首次接触 DocMee Open Platform 的团队先理解平台的能力边界，再判断应该优先走 `UI 接入`、`API 接入` 还是 `开放能力`。

## 平台能做什么

- 通过 `UI 接入` 把完整的 AI PPT 创作、模板、编辑和下载体验嵌入到业务系统
- 通过 `API 接入` 控制任务创建、内容生成、模板处理、PPT 下载与管理链路
- 通过 `开放能力` 获取特定能力模块，例如 HTML 转 PPTX 这类单点能力
- 通过模板、PPT、记录和定价相关页面补齐完整的交付与运营链路

## 能力边界

DocMee 更适合承担“演示内容生产平台”角色，而不是单纯的静态模板下载站。它擅长的是：

- 让业务系统快速获得 AI PPT 创作能力
- 提供可编排的 PPT 生成链路和中间结果
- 支持模板、文件、记录、下载等配套能力

如果你的目标只是做一个完全自定义的前端编辑器，或者只需要少量静态模板展示，DocMee 并不是最直接的选择。

## 如何选择接入方式

### 什么时候选 UI 接入

优先选择 [UI 接入](/ui-integration) 的场景：

- 你希望快速嵌入完整工作台，而不是自己搭建整套前端体验
- 你需要创作页、列表页、编辑页和模板页的一体化能力
- 你更关注集成速度和完整体验，而不是逐个控制底层接口

### 什么时候选 API 接入

优先选择 [快速开始](/getting-started) 和 [API 参考](/api-reference) 的场景：

- 你需要在服务端完全控制任务链路和中间结果
- 你需要把生成能力编排进现有业务流程
- 你需要按接口粒度管理模板、PPT、记录和下载逻辑

### 什么时候选开放能力

优先选择 [开放能力](/open-capabilities) 的场景：

- 你只需要某个特定能力，而不是整套平台能力
- 你希望从专项能力快速验证可行性
- 你要处理类似 [HTML 转 PPTX](/open-capabilities/html-to-pptx/introduction) 这样的单点能力

1. 先阅读本页，确定你的目标更接近 `UI 接入`、`API 接入` 还是 `开放能力`
2. 要嵌入完整工作台时，进入 [UI 接入](/ui-integration)

# 最佳实践

## 请求前自检 HTML

- 在业务侧提交前先校验每页是否包含 <InlineCode>.slide</InlineCode> 根元素，详细限制见 [❗限制条件](/open-capabilities/html-to-pptx/integration-notes)。
- 在业务侧优先将 <InlineCode>.slide</InlineCode> 控制为 <InlineCode>1280 × 720</InlineCode>，并统一页面宽高，避免混入非同尺寸页面；尺寸规则见 [❗限制条件](/open-capabilities/html-to-pptx/integration-notes)。
- 对生成后的 HTML 做基础渲染检查，避免明显缺失内容后再提交转换。

- 如果你的业务要求页数稳定，建议使用 <InlineCode>blank</InlineCode>；参数限制见 [❗限制条件](/open-capabilities/html-to-pptx/integration-notes)。

# 能力计费

HTML 转 PPTX 按单次转换任务的成功转换页数计费。

## 计费规则

1. 每一个成功转换的页面消耗 `0.3` 积分。
2. 当单次任务的成功转换页数大于等于 `35` 页时，按 `1.2` 倍计算。
3. 当单次任务的成功转换页数大于等于 `50` 页时，按 `1.6` 倍计算。
4. 单次任务最终消耗积分向下取整。
5. 转换失败的页面不计入消耗积分。

## 结算公式

```text
消耗积分 = 向下取整(成功转换页数 × 0.3 × 倍率)
```

其中倍率规则如下：

| 成功转换页数 | 倍率 |
| --- | --- |
| `< 35` | `1.0` |
| `>= 35` 且 `< 50` | `1.2` |
| `>= 50` | `1.6` |

## 计费示例

1. 如果成功转换页数为 `20` 页，则消耗积分 = `20 × 0.3 × 1.0 = 6`，向下取整后为 `6` 积分。
2. 如果成功转换页数为 `35` 页，则消耗积分 = `35 × 0.3 × 1.2 = 12.6`，向下取整后为 `12` 积分。
3. 如果成功转换页数为 `50` 页，则消耗积分 = `50 × 0.3 × 1.6 = 24`，向下取整后为 `24` 积分。

## 说明

- 接口返回的 `successPages` 可用于核对本次任务的实际计费页数。
- 若任务存在失败页面，仅对成功转换的页面计费。

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `creatorData` | `object` | 否 | 从业务侧直接带入创作数据 |
| `creatorMode` | `('topic' \| 'material')[]` | 否 | 控制创建页允许的生成方式 |
| `outlineExportFormat` | `'md' \| 'txt'` | 否 | 导出大纲的默认文件格式 |
| `templateMenu` | `('system' \| 'custom' \| 'used')[]` | 否 | 模板面板默认展示的模板类型 |
| `createCustomTemplateWhenEmpty` | `boolean` | 否 | 自定义模板为空时是否显示“立即创建” |

- `outlineGenerated`
- `beforeCreatePpt`
- `charge`
- `afterGenerate`
- `beforeDownload`

- `automaticSavePPT`

- `pptxRenamed`

## 核心事件

### mounted

iframe 挂载完成后触发：

```js
{
  type: 'mounted'
}
```

### beforeGenerate

用户开始生成前触发。你可以在这里做额度校验、风控或补充业务确认。

```js
{
  type: 'beforeGenerate',
  data: {
    subtype: 'outline', // 或 'ppt'
    fields: {}
  }
}
```

该事件允许返回：

- `true`：继续生成
- `false`：阻止生成
- `Promise<boolean>`：异步决定是否继续

### beforeCreatePpt

新版 V2 在选完模板、真正开始生成 PPT 前，还会多触发一个拦截点：

```js
{
  type: 'beforeCreatePpt',
  data: {
    taskId: 'task_xxx',
    templateId: 'template_xxx',
    outline: 'markdown content',
    byCommand: true
  }
}
```

该事件同样支持返回 `true / false / Promise<boolean>`。从当前实现看，如果宿主侧没有及时回复，该拦截默认会在约 `500ms` 后放行。

### pageChange

页面切换时触发：

```js
{
  type: 'pageChange',
  data: {
    page: 'creator'
  }
}
```

当前实际发送的 `page` 值可能包括：

- `dashboard`
- `editor`
- `customTemplate`
- `creator`
- `creatorV2`
- `templateCreator`
- `templateMarker`

### toggleGenerateMode

创建页切换“主题生成 / 导入资料”以及资料子模式时触发：

```js
{
  type: 'toggleGenerateMode',
  data: {
    mode: 'subject',
    subMode: undefined
  }
}
```

其中 `subMode` 在资料模式下可能是：

- `text`
- `file`
- `website`
- `outline`

### outlineGenerated

当新版创建流程已经产出结构化大纲对象时触发：

```js
{
  type: 'outlineGenerated',
  data: {
    outline: {}
  }
}
```

### beforeDownload

下载前触发，可用于拦截权限，也可以返回自定义文件名：

```js
  data: {
    id: 'ppt_xxx',
    subject: '季度经营汇报'
  }
}
```

该事件允许返回：

- `true`：允许下载
- `false`：阻止下载
- `string`：自定义文件名，建议以 `.pptx` 结尾
- `Promise<boolean | string>`：异步决定

### charge

PPT 生成完成并发生扣费时触发：

```js
{
  type: 'charge',
  data: {
    id: 'ppt_xxx'
  }
}
```

### afterGenerate

PPT 成功生成后触发：

```js
{
  type: 'afterGenerate',
  data: {
    id: 'ppt_xxx'
  }
}
```

错误事件统一从这里接收：

```js
  type: 'error',

  }
}
```

从当前实现看，`error.data` 有两种常见形态：

- `{ code, message }`
- `{ error }`

```js
{
  type: 'user-info',

    uid: 'user_xxx',

    usedCount: 2
  }
}
```

当 `token` 校验失败时触发：

```js
{
  type: 'invalid-token'
}
```

### beforeCreateCustomTemplate

创建完整自定义模板前触发：

```js

  type: 'beforeCreateCustomTemplate',
  data: {
    file: {},
    totalPptCount: 99
  }
}
```

该事件支持返回 `true / false / Promise<boolean>`。

自定义模板创建或修改完成后触发：

用户主动点击保存时触发：

```js
    id: 'ppt_xxx',
```

## 典型业务处理示例

```typescript
docmee.on('beforeGenerate', async ({ subtype, fields }) => {
  const allowed = await fetch('/api/quota/check', {
    method: 'POST',
    body: JSON.stringify({ subtype, fields }),
  }).then((res) => res.json())

  return allowed.ok
})

docmee.on('beforeDownload', ({ subject }) => {
  return `Docmee_${subject}.pptx`
})

docmee.on('error', ({ code, message }) => {
  console.error(code, message)
})
```

## 事件设计建议

- 在 `beforeGenerate` 中做额度和风控判断，不要等生成后再回滚
- 如果你需要在“选完模板但还没真正出 PPT”时做最后一次校验，拦截 `beforeCreatePpt`
- 在 `beforeDownload` 中统一做命名规范和权限控制
- 对 `error` 里的业务码做分类处理，尤其是次数耗尽、鉴权失效和参数错误
- 需要感知编辑器内保存动作时，优先监听 `manuallySavePPT` 和 `automaticSavePPT`
- 如果宿主侧要跟随编辑器页切换或标题变更，监听 `changeSlideIndex` 和 `pptxRenamed`

# 实例方法

以下方法都属于 <InlineCode>DocmeeUI</InlineCode> 实例成员，通常通过 <InlineCode>docmee.xxx()</InlineCode> 调用。

## 常用方法一览

- `updateToken(newToken: string)`：更新用户 Token
- `destroy()`：卸载 iframe
- `getInfo()`：主动拉取一次用户信息，结果会通过事件返回
- `navigate({ page, pptId, templateId })`：在创建页、列表页、编辑页和自定义模板页之间跳转
- `sendMessage({ type, content })`：向 SDK 发送提示消息
- `changeCreatorData(data, createNow?)`：在创建页中动态修改输入内容
- `updateTemplate(templateId)`：外部指定切换模板并刷新
- `showTemplateDialog(type?)`：弹出模板选择弹窗
- `getCurrentPptInfo()`：主动获取当前 PPT 信息
- `importCSS(css)`：运行时追加 CSS
- `continueCreatePpt()`：宿主侧确认后继续创建 PPT
- `changeSlidePageIndex(pageIndex)`：外部切换到指定页
- `changeSubjectContent(content)`：修改创建页主题/正文输入
- `submitNewCreator()`：触发新版创建器提交
- `reloadEditor()`：重新加载当前编辑器数据

## 页面跳转

```typescript
docmee.navigate({ page: 'dashboard' })
docmee.navigate({ page: 'creator' })
docmee.navigate({ page: 'editor', pptId: 'ppt_xxx' })
docmee.navigate({ page: 'templateMarker', templateId: 'template_xxx' })
```

注意：

- 跳到 `editor` 时必须传 `pptId`
- 跳到 `templateMarker` 时应传 `templateId`

```typescript
docmee.updateToken('new_token')
```

```typescript
docmee.sendMessage({
```

# 快速开始

## 最小接入步骤

1. 服务端调用 [createApiToken](/api-reference/auth/create-api-token) 生成用户级 <InlineCode>token</InlineCode>
2. 前端安装 <InlineCode>@docmee/sdk-ui</InlineCode>
3. 在页面中实例化 <InlineCode>DocmeeUI</InlineCode>，并传入挂载容器、页面类型和 <InlineCode>token</InlineCode>
4. 根据业务需要监听生成、跳页、下载和错误事件

## NPM 方式接入

```bash
npm install @docmee/sdk-ui
```

### TypeScript 示例

```typescript
import { CreatorType, DocmeeUI } from '@docmee/sdk-ui'

const docmee = new DocmeeUI({
  container: 'app',
  page: 'creator',
  creatorVersion: 'v2',
  token: 'your_token',
  lang: 'zh',
  mode: 'light',
  creatorData: {
    type: CreatorType.AI_GEN,
    subject: 'AI 行业未来 10 年的发展预测',
  },
})

docmee.on('mounted', () => {
  console.log('UI mounted')
})

docmee.on('beforeGenerate', (payload) => {
  console.log('beforeGenerate', payload)
  return true
})

docmee.on('pageChange', (payload) => {
  console.log('pageChange', payload)
})
```

## 让创建页直接带入业务数据

```typescript
const docmee = new DocmeeUI({
```
