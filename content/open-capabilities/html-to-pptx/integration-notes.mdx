# ❗限制条件

本页只说明接入前必须明确的约束，不展开完整接口契约。

## 鉴权方式

- 调用接口时在请求头中传入 <InlineCode>token</InlineCode>  [获取Token](/api-reference/auth/create-api-token)
- 请求体类型为 <InlineCode>application/json</InlineCode>

## HTML 输入约束

- 每个 HTML 都必须包含一个 <InlineCode>.slide</InlineCode> 根元素，系统按该元素识别单页内容。
- <InlineCode>.slide</InlineCode> 类名元素的目标尺寸应为 <InlineCode>1280 × 720</InlineCode>。
- 如果 <InlineCode>.slide</InlineCode> 不是 <InlineCode>1280 × 720</InlineCode>，系统仍会尝试转换，并尽力缩放到目标尺寸，但最终排版与视觉效果不保证。
- 同一次请求中的所有 HTML 页面应保持同一尺寸，避免混入不同尺寸页面影响转换效果。
- <InlineCode>htmls</InlineCode> 数组顺序就是最终 PPT 的页顺序。
- <InlineCode>htmls</InlineCode> 中每个元素都必须是字符串。

## 文件命名约定

- <InlineCode>filename</InlineCode> 是可选参数，用于指定最终下载文件时的 PPTX 文件名。
- 如果不传 <InlineCode>filename</InlineCode>，系统会使用默认文件名。
- 如果传入的 <InlineCode>filename</InlineCode> 不带 <InlineCode>.pptx</InlineCode> 后缀，服务端会自动补全。
- <InlineCode>filename</InlineCode> 只影响最终下载文件名，不影响 HTML 转换逻辑和页级处理规则。

## 失败页处理约束

- <InlineCode>pageErrorMode</InlineCode> 只支持 <InlineCode>blank</InlineCode> 和 <InlineCode>skip</InlineCode>。
- 默认值是 <InlineCode>blank</InlineCode>。
- <InlineCode>blank</InlineCode> 表示失败页在结果文件中保留为空白页。
- <InlineCode>skip</InlineCode> 表示失败页直接跳过，不进入最终文件。

## 下载与保存约束

<ImportantNote title="下载地址说明">
  <p>
    响应中的 <InlineCode>fileUrl</InlineCode> 是临时 OSS 下载地址，存在失效时间。请在 <InlineCode>expireAt</InlineCode> 前尽快下载或转存。
  </p>
  <p>
    平台不提供长期保存，也不建议将该地址作为永久资源链接使用。
  </p>
</ImportantNote>

## 接入前建议

- 调用前先校验 HTML 是否包含 <InlineCode>.slide</InlineCode> 根元素。
- 调用前优先保证所有页面都使用 <InlineCode>1280 × 720</InlineCode>，并在同一次请求中保持统一尺寸。
- 如果业务对页数稳定性要求高，优先考虑 <InlineCode>pageErrorMode = blank</InlineCode>。
- 如果业务更关注仅保留成功页面，可考虑 <InlineCode>pageErrorMode = skip</InlineCode>。
