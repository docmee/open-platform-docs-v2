# 生成第一份 AI 智能布局 PPT

这一节只走一条最短路径：直接生成 `JSON` 结构化大纲，再调用 AIL 接口完成页面级智能布局。

## 这条链路和模板模式有什么不同

- 模板模式的核心输入是 `Markdown` 大纲
- AI 智能布局的核心输入是 `JSON` 结构化大纲
- AIL 返回的是分页 `HTML` 流和任务进度，适合边生成边展示、断流恢复和最终状态确认


## 第 1 步：创建调用 Token

先在服务端使用 `Api-Key` 调用 [`createApiToken`](/api-reference/auth/create-api-token)。

<ApiCodeTabs items={['Node.js', 'Python', 'Go', 'Java']}>
  <ApiCodeTab>

```js
const response = await fetch('https://docmee.cn/api/user/createApiToken', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Api-Key': 'YOUR_API_KEY'
  },
  body: JSON.stringify({
    uid: 'demo-user',
    limit: 10,
    timeOfHours: 2
  })
})

const result = await response.json()
console.log(result)
```

这个接口会以 `SSE` 形式持续返回两类消息：

- 过程消息：当前步骤、分页任务和整体进度

如果连接持续正常，前端可以一边收流，一边实时渲染页面。

常见状态值如下：

- `0`：等待处理
- `1`：处理中
- `2`：处理完成
- `3`：处理失败

## 一次完整闭环

把整条链路压缩成一组最小调用，就是：

1. `createApiToken`
2. `createTask`
3. `generateContent`，并传 `outlineType=JSON`
4. `generatePptxByAi`
5. 断流时调用 `latestData`
6. 最终调用 `getConvertResult`，直到状态为 `2`
