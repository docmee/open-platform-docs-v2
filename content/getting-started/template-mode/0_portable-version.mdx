# 生成第一份 PPT

这一节介绍便携版接口：把 `V2` 的“创建任务、生成内容、生成 PPT、下载文件”整合成一套更容易接入的上层能力。

如果你只想接受用户输入，然后异步拿到最终 PPT 文件，优先接入便携版；如果你需要自己控制内容生成、模板切换和二次编辑流程，请使用[完整版](/getting-started/first-ppt/1_complete-version)。

## 便携版接口设计

便携版默认只暴露 2 个接口：

- `POST /api/ppt/portable/createTask`：创建 PPT 生成任务
- `POST /api/ppt/portable/taskResult`：查询任务结果

服务端内部会自动串联以下标准能力：

- 创建任务
- 生成大纲与正文
- 套用指定模板
- 生成 PPT
- 生成下载链接

也就是说，接入方不再需要分别调用 `generateContent`、`generatePptx` 和 `downloadPptx`。

## 第 1 步：创建调用 Token

便携版仍然沿用现有鉴权方式。先在服务端使用 `Api-Key` 调用 [`createApiToken`](/api-reference/auth/create-api-token)。

拿到返回的 `token` 后，后续请求都在 Header 中传入：

```http
token: YOUR_TEMP_TOKEN
```

## 第 2 步：创建便携版任务

调用 `POST /api/ppt/portable/createTask` 创建一个异步任务。

### 推荐请求字段

- `prompt`：用户输入的主题或完整生成要求，必填
- `templateId`：模板 ID，必填
- `fileName`：导出文件名，可选
- `lang`：生成语言，可选，默认 `zh`
- `pages`：页数偏好，可选
- `scene`：生成场景，可选，例如“产品介绍”“方案汇报”
- `audience`：受众，可选，例如“客户”“管理层”
- `tone`：语气风格，可选，例如“专业”“简洁”
- `ratio`：页面比例，可选，例如 `16:9`
- `images`：是否偏好多图，可选，布尔值
- `references`：补充资料，可选，字符串数组

### 返回字段

- `taskId`：任务 ID
- `status`：初始状态，通常为 `pending`
- `message`：创建结果说明

## 第 3 步：轮询任务结果

调用 `POST /api/ppt/portable/taskResult` 查询任务进度与最终结果。

### 请求字段

- `taskId`：第 2 步返回的任务 ID，必填

### 返回字段

- `taskId`：任务 ID
- `status`：任务状态，`pending`、`processing`、`success`、`failed`
- `progress`：任务进度，可选，范围 `0-100`
- `step`：当前处理阶段，可选
- `pptId`：成功时返回
- `fileUrl`：成功时返回，PPT 下载地址
- `coverUrl`：成功时返回，封面图地址
- `previewUrl`：成功时返回，预览地址
- `errorCode`：失败时返回
- `errorMessage`：失败时返回

## 简易版与完整版的区别

简易版适合下面这类场景：

- 你已经知道用户的输入长什么样，只想快速输出 PPT
- 你不需要把“大纲生成”和“PPT 生成”拆成多个交互步骤
- 你希望后端统一编排标准版接口，前端只维护最少逻辑

如果你需要以下能力，请使用[完整版](/getting-started/first-ppt/1_complete-version)：

- 先生成 Markdown，再让用户手动编辑后继续生成
- 先让用户挑选多套模板，再决定最终导出
- 对不同阶段分别做状态控制、内容干预或重试

## 常见建议

- 新接入优先用便携版，复杂场景再切换到完整版
- 生产环境由服务端创建 Token，不要在前端暴露 `Api-Key`
- 建议业务侧保存 `taskId`，便于失败重试和结果查询
- `templateId` 建议通过[`templates`](/api-reference/templates/list-templates)或随机模板接口预先获取
- `fileUrl` 建议视为临时下载链接，具体有效期以服务端配置为准
