# 直接生成 PPT

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/api/ppt/directGeneratePptx">https://open.docmee.cn/api/ppt/directGeneratePptx</InlineCode>
- 鉴权：请求头传 `token`

## 接口作用

一步式直接生成 PPT，适合快速体验或不想拆成大纲与内容两个阶段的场景。

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/api/ppt/directGeneratePptx \
  --header "Content-Type: application/json" \
  --header "token: YOUR_TEMP_TOKEN" \
  --data '{
  "stream": false,
  "templateId": "template_xxx",
  "pptxProperty": false,
  "length": "medium",
  "lang": "zh",
  "prompt": "风格更专业",
  "subject": "AI 未来发展",
  "dataUrl": "https://xxx"
}'
```

## 请求参数

```json
{
  "stream": false,
  "templateId": "template_xxx",
  "pptxProperty": false,
  "length": "medium",
  "lang": "zh",
  "prompt": "风格更专业",
  "subject": "AI 未来发展",
  "dataUrl": "https://xxx"
}
```


## 请求字段说明

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `stream` | `boolean` | 否 | `false` | 是否启用流式返回。 |
| `templateId` | `string` | 否 | `template_xxx` | 模板 ID。 |
| `pptxProperty` | `boolean` | 否 | `false` | 字段用途请结合接口场景传入。 |
| `length` | `string` | 否 | `medium` | 字段用途请结合接口场景传入。 |
| `lang` | `string` | 否 | `zh` | 输出语言。 |
| `prompt` | `string` | 否 | `风格更专业` | 附加生成要求。 |
| `subject` | `string` | 否 | `AI 未来发展` | 字段用途请结合接口场景传入。 |
| `dataUrl` | `string` | 否 | `https://xxx` | 字段用途请结合接口场景传入。 |

## 说明

- `subject` 与 `dataUrl` 可任选其一，也可同时提供
- 非流式返回时会直接给出 `pptInfo`
- 初期验证能力时可以使用，但新正式接入仍推荐 `V2`


## 响应示例

```text
event: message
data: {"status":3,"text":"生成中..."}

event: message
data: {"status":4,"markdown":"# 主题"}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `event` | `string` | `message` | SSE 事件名。 |
| `data.status` | `number` | `3` | 当前生成状态。常见值如 1、3、4。 |
| `data.text` | `string` | `生成中...` | 当前阶段返回的文本片段或提示。 |
| `data.markdown` | `string` | `# 主题` | 生成完成后返回的 Markdown 内容。 |
