# 修改大纲（V1）

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/ppt/updateOutline">https://open.docmee.cn/ppt/updateOutline</InlineCode>
- 鉴权：请求头传 `token`

## 接口作用

`V1` 兼容接口。根据用户指令重新组织现有大纲结构。

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/ppt/updateOutline \
  --header "Content-Type: application/json" \
  --header "token: YOUR_TEMP_TOKEN" \
  --data '{
  "outlineMarkdown": "# 主题\n## 章节\n### 页面标题\n#### 内容标题",
  "length": "medium",
  "question": "让大纲更专业"
}'
```

## 请求参数

```json
{
  "outlineMarkdown": "# 主题\n## 章节\n### 页面标题\n#### 内容标题",
  "length": "medium",
  "question": "让大纲更专业"
}
```


## 请求字段说明

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `outlineMarkdown` | `string` | 否 | `# 主题 / ## 章节 / ### 页面标题 / #### 内容标题` | Markdown 格式的内容结构。 |
| `length` | `string` | 否 | `medium` | 字段用途请结合接口场景传入。 |
| `question` | `string` | 否 | `让大纲更专业` | 修改意见或追问指令。 |

## 使用说明

- `question` 可传系统内置优化指令，也可传用户自定义要求
- 返回为流式结构，响应格式与 [`generateOutline`](/api-reference/v1/generate-outline) 一致


## 响应示例

```text
event: message
data: {"status":3,"text":"生成中..."}

event: message
data: {"status":4,"markdown":"# 主题"}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `event` | `string` | `message` | SSE 事件名。 |
| `data.status` | `number` | `3` | 当前生成状态。常见值如 1、3、4。 |
| `data.text` | `string` | `生成中...` | 当前阶段返回的文本片段或提示。 |
| `data.markdown` | `string` | `# 主题` | 生成完成后返回的 Markdown 内容。 |
