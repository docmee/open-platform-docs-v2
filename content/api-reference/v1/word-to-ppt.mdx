# Word 转 PPT

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/api/ppt/v1/word2pptx">https://open.docmee.cn/api/ppt/v1/word2pptx</InlineCode>
- 鉴权：请求头传 `token`
- 请求体：`multipart/form-data`

## 接口作用

尽可能保留 Word 文件原有层级结构和内容表达，直接生成 PPT。

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/api/ppt/v1/word2pptx \
  --header "token: YOUR_TEMP_TOKEN" \
  --form "file=@demo.docx" \
  --form "templateId=TEMPLATE_ID" \
  --form "stream=VALUE"
```

## 请求参数

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `file` | `File` | `docx` 文件，小于 30M |
| `templateId` | `string` | 模板 ID，可空 |
| `stream` | `boolean` | 是否流式返回 |

## 请求字段说明

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `file` | `File` | 是 | `demo.docx` | 需要转换的 Word 文件，建议小于 30M。 |
| `templateId` | `string` | 否 | `template_xxx` | 指定用于生成 PPT 的模板。 |
| `stream` | `boolean` | 否 | `false` | 是否使用流式返回。未传时通常按非流式结果处理。 |

## 响应说明

- 非流式模式下返回 `pptInfo`
- 可直接拿到生成后的 PPT 文件和结构数据


## 响应示例

```text
event: message
data: {"status":3,"text":"生成中..."}

event: message
data: {"status":4,"markdown":"# 主题"}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `event` | `string` | `message` | SSE 事件名。 |
| `data.status` | `number` | `3` | 当前生成状态。常见值如 1、3、4。 |
| `data.text` | `string` | `生成中...` | 当前阶段返回的文本片段或提示。 |
| `data.markdown` | `string` | `# 主题` | 生成完成后返回的 Markdown 内容。 |
