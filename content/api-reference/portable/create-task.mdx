# 创建便携版任务

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/v2/api/ppt/portable/create-task">https://open.docmee.cn/v2/api/ppt/portable/create-task</InlineCode>
- 鉴权：请求头传 <InlineCode>token</InlineCode>
- 请求体：支持 <InlineCode>application/json</InlineCode> 和 <InlineCode>multipart/form-data</InlineCode>

## 接口作用

创建一个便携版 PPT 生成任务。服务端会在内部串联标准版的创建任务、生成内容、生成 PPT 和生成下载地址流程，适合希望“一次发起，统一拿结果”的场景。

<ImportantNote title="重要信息">
  <p>
    当 <InlineCode>stream=true</InlineCode> 时，接口会直接以 <strong>SSE</strong> 方式返回过程数据；当 <InlineCode>stream=false</InlineCode> 时，
    先返回 <InlineCode>taskId</InlineCode>，再通过 <InlineCode>taskResult</InlineCode> 查询最终结果。上传文件场景请使用 <InlineCode>multipart/form-data</InlineCode>。
  </p>
</ImportantNote>

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/v2/api/ppt/portable/create-task \
  --header "Content-Type: application/json" \
  --header "token: YOUR_TEMP_TOKEN" \
  --data '{
    "type": 1,
    "content": "请生成一份关于 AI 办公趋势的产品介绍 PPT",
    "templateId": "TEMPLATE_ID",
    "stream": false,
    "length": "medium",
    "scene": "产品介绍",
    "audience": "客户",
    "lang": "zh",
    "prompt": "语气专业，适合演示"
  }'
```

## 请求参数

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `type` | `number` | 是 | `1` | 任务类型。支持智能生成、上传文件、网页链接、文本粘贴、Markdown 大纲等入口。 |
| `content` | `string` | 否 | `请生成一份关于 AI 办公趋势的产品介绍 PPT` | 根据 `type` 传主题、网页 URL、文本内容或 Markdown。 |
| `file` | `File[]` | 否 | `demo.docx` | 上传文件场景使用，仅 `multipart/form-data` 支持。 |
| `templateId` | `string` | 是 | `TEMPLATE_ID` | 最终生成 PPT 使用的模板 ID。 |
| `stream` | `boolean` | 否 | `false` | 是否使用 SSE 返回生成过程，默认 `false`。 |
| `length` | `string` | 否 | `medium` | 内容长度偏好，常见值为 `short`、`medium`、`long`。 |
| `scene` | `string` | 否 | `产品介绍` | 演示场景。 |
| `audience` | `string` | 否 | `客户` | 受众类型。 |
| `lang` | `string` | 否 | `zh` | 输出语言。 |
| `prompt` | `string` | 否 | `语气专业，适合演示` | 追加生成要求，建议控制在 50 字以内。 |

## 参数说明

- <InlineCode>application/json</InlineCode> 适用于不上传文件的场景。
- <InlineCode>multipart/form-data</InlineCode> 适用于上传本地文件的场景。
- <InlineCode>prompt</InlineCode> 通常只在 <InlineCode>type=1</InlineCode>、<InlineCode>2</InlineCode>、<InlineCode>5</InlineCode>、<InlineCode>6</InlineCode> 时生效。
- <InlineCode>type=2</InlineCode>、<InlineCode>type=4</InlineCode> 场景下通常通过 <InlineCode>file</InlineCode> 上传原始文件。

## 响应示例

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "taskId": "portable_task_xxx",
    "status": "pending"
  }
}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `code` | `number` | `0` | 通用响应状态码。 |
| `message` | `string` | `操作成功` | 响应说明文本。 |
| `data` | `object` | `{"taskId":"portable_task_xxx"}` | 便携版任务的业务数据。 |
| `data.taskId` | `string` | `portable_task_xxx` | 便携版任务 ID，用于后续轮询结果。 |
| `data.status` | `string` | `pending` | 当前任务状态。常见值包括 `pending`、`processing`、`success`、`failed`。 |

import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class PortableCreateTaskExample {
  public static void main(String[] args) throws Exception {
    String body = """
      {
        "type": 1,
        "content": "请生成一份关于 AI 办公趋势的产品介绍 PPT",
        "templateId": "TEMPLATE_ID",
        "stream": false,
        "length": "medium",
        "scene": "产品介绍",

        "lang": "zh",
        "prompt": "语气专业，适合演示"
      }
      """;

    HttpRequest request = HttpRequest.newBuilder()
      .uri(URI.create("https://open.docmee.cn/v2/api/ppt/portable/create-task"))
      .header("Content-Type", "application/json")
      .header("token", "YOUR_TEMP_TOKEN")
      .POST(HttpRequest.BodyPublishers.ofString(body))
      .build();

    HttpResponse<String> response = HttpClient.newHttpClient()
      .send(request, HttpResponse.BodyHandlers.ofString());

    System.out.println(response.body());
  }
}
