# 保存 PPT

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/api/ppt/savePptx">https://open.docmee.cn/api/ppt/savePptx</InlineCode>
- 鉴权：请求头传 `token`

## 接口作用

保存编辑后的 PPT 结构，并按需重新渲染 PPT 文件和封面。

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/api/ppt/savePptx \
  --header "Content-Type: application/json" \
  --header "token: YOUR_TEMP_TOKEN" \
  --data '{
  "id": "ppt_xxx",
  "drawPptx": true,
  "drawCover": true,
  "pptxProperty": {}
}'
```

## 请求参数

```json
{
  "id": "ppt_xxx",
  "drawPptx": true,
  "drawCover": true,
  "pptxProperty": {}
}
```


## 请求字段说明

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | `string` | 否 | `ppt_xxx` | 业务对象 ID。 |
| `drawPptx` | `boolean` | 否 | `true` | 字段用途请结合接口场景传入。 |
| `drawCover` | `boolean` | 否 | `true` | 字段用途请结合接口场景传入。 |
| `pptxProperty` | `object` | 否 | `{}` | 字段用途请结合接口场景传入。 |

## 说明

- 只想重渲染 PPT 文件时，可只传 `drawPptx=true`
- 如果没有最新的 `pptxProperty`，可不传该字段
- 成功后会返回最新的 `pptInfo`


## 响应示例

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    "id": "VALUE"
  }
}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `code` | `number` | `0` | 通用响应状态码，0 表示成功。 |
| `message` | `string` | `操作成功` | 服务端返回的结果说明。 |
| `data` | `object` | `{...}` | 业务数据对象。 |
