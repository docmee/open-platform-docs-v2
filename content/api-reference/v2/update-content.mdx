# 修改大纲内容

## 接口信息

- 方法：<HttpMethodBadge method="POST" />
- 路径：<InlineCode value="https://open.docmee.cn/api/ppt/v2/updateContent">https://open.docmee.cn/api/ppt/v2/updateContent</InlineCode>
- 鉴权：请求头传 `token`

## 接口作用

基于已有 Markdown 内容和用户修改意见，重新生成任务内容，适合做“继续优化”“改成另一种结构”等二次编辑。

## 请求示例

```bash
curl --request POST \
  --url https://open.docmee.cn/api/ppt/v2/updateContent \
  --header "Content-Type: application/json" \
  --header "token: YOUR_TEMP_TOKEN" \
  --data '{
  "id": "task_xxx",
  "stream": true,
  "markdown": "# 主题\n## 章节\n### 页面标题\n#### 内容标题\n- 内容",
  "question": "把结构改得更像金字塔原则"
}'
```

## 请求参数

```json
{
  "id": "task_xxx",
  "stream": true,
  "markdown": "# 主题\n## 章节\n### 页面标题\n#### 内容标题\n- 内容",
  "question": "把结构改得更像金字塔原则"
}
```

- `id`：任务 ID
- `stream`：是否流式返回
- `markdown`：当前大纲与内容
- `question`：修改意见


## 请求字段说明

| 参数名 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | `string` | 否 | `task_xxx` | 业务对象 ID。 |
| `stream` | `boolean` | 否 | `true` | 是否启用流式返回。 |
| `markdown` | `string` | 否 | `# 主题 / ## 章节 / ### 页面标题 / #### 内容标题 / - 内容` | Markdown 格式的内容结构。 |
| `question` | `string` | 否 | `把结构改得更像金字塔原则` | 修改意见或追问指令。 |

## 响应方式

支持 `event-stream` 和 `application/json`，整体结构与 [`generateContent`](/api-reference/v2/generate-content) 一致。


## 响应示例

```text
event: message
data: {"status":3,"text":"生成中..."}

event: message
data: {"status":4,"markdown":"# 主题"}
```

## 响应字段说明

| 字段名 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| `event` | `string` | `message` | SSE 事件名。 |
| `data.status` | `number` | `3` | 当前生成状态。常见值如 1、3、4。 |
| `data.text` | `string` | `生成中...` | 当前阶段返回的文本片段或提示。 |
| `data.markdown` | `string` | `# 主题` | 生成完成后返回的 Markdown 内容。 |
