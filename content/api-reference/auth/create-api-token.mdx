# 创建接口 Token

## 接口信息

- 方法：`POST`
- 路径：`/api/user/createApiToken`
- 鉴权：请求头传 `Api-Key`

## 接口作用

创建后续 API 调用使用的临时 `token`。该 Token 支持按 `uid` 做数据隔离，也支持通过 `limit` 限制最多可生成的 PPT 次数。

## 请求参数

```json
{
  "uid": "third-party-user-id",
  "limit": 10,
  "timeOfHours": 2
}
```

- `uid`：第三方用户 ID，建议传入，便于用户隔离
- `limit`：允许生成 PPT 的次数；`0` 表示不能生成
- `timeOfHours`：有效期小时数，默认 2，最大 48

## 响应示例

```json
{
  "data": {
    "token": "sk_xxx",
    "expireTime": 7200
  },
  "code": 0,
  "message": "操作成功"
}
```

## 使用说明

- 该接口应只在服务端调用
- 同一个 `uid` 再次创建 Token 时，旧 Token 会在约 10 秒内过期
- UI 集成场景建议始终设置 `limit`，降低泄露风险
